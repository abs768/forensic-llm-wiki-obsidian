# Obsidian Vault — `case_002_evolving`

This folder is an export of the compiled investigation wiki for
`case_002_evolving`. It is plain markdown — open this folder as an Obsidian
vault and the graph view will render the wiki-link relationships
between pages.

## Files

- `index.md`
- `timeline.md`
- `entities.md`
- `iocs.md`
- `hypotheses.md`
- `contradictions.md`
- `open_questions.md`
- `final_report.md`
- `graph.mmd`

## How to use this vault

1. Open Obsidian and choose "Open folder as vault" → point at this directory.
2. The graph view shows every page and how `[[wiki link]]` references
   connect them.
3. `index.md` is the entry point. From there, follow `[[timeline]]`,
   `[[entities]]`, `[[iocs]]`, `[[hypotheses]]`, `[[contradictions]]`,
   `[[open_questions]]`, `[[final_report]]`.
4. `graph.mmd` (if present) is the GraphRAG-lite relationship diagram.
   GitHub and many Obsidian Mermaid plugins render it natively.

## What you should NOT edit here

- `raw_sources/` lives elsewhere in the project. It is **immutable**.
  Do not edit raw evidence in Obsidian.
- The `.fw/` sidecar (state, indexes, traces) is not included in this
  export. Don't try to round-trip it from Obsidian.

## Reviewing AI-proposed changes

The project ships a human review queue. Any wiki update that contains
a risky phrase (e.g. "confirmed malware") is held back from being
applied until a human approves or rejects it. See `docs/human_review.md`
in the project repo.

This vault is generated by `python fw.py export-obsidian case_002_evolving`.
